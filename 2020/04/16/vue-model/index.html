:model与v-model

发表于 2020-04-16 | 分类于 前端 | 浏览 次

最近在做毕设，正好碰到:model与v-model的问题，就想着写一下这两者的区别。

:model

:model是v-bind:model的缩写,这种只是将父组件的数据传递到了子组件，并没有实现子组件和父组件数据的双向绑定。当然引用类型除外，子组件改变引用类型的数据的话，父组件也会改变的。

上述代码就是把input里面的值与data里面的index的值绑定起来，当index的变化时input里面的值也会随着变化，而input的值不能变化也就是说此时input失去了他的输入作用。

V-model

v-model主要是用在表单元素中，它实现了双向绑定。就是父组件的值改变时，子组件的值也会改变，子组件的值改变，父组件也会跟着改变，也就相当于此时的父子组件从同一个地址里面拿东西，放东西。

还是刚才那串代码，此时的:model已经改成了v-model，此时不光index的变化时input里面的值也会随着变化，而当input的值变化时，index的值也会跟着变化，可以直接把index的值提交或操作。
